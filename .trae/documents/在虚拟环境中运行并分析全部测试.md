## 任务目标
- 使用 conda 激活“qubo-prover”环境，严格不进行任何依赖安装。
- 运行全部测试，若环境依赖异常，立即停止并报告。
- 将测试统计与分析（含真实失败用例）整理成文档。

## 任务点与顺序
1. 激活环境
- 执行：`conda activate qubo-prover`（仅环境切换）。
- 验证：输出当前 Python 路径与版本，确认处于目标环境。

2. 基本环境检查
- 执行：
  - `python -V`
  - `python -c "import sys; print(sys.executable)"`
  - `pip -V`
  - `python -c "import pyqubo, dimod; print('pyqubo ok, dimod ok')"`
- 判定：任一模块导入失败或版本异常，立即停止并报告，不进行任何安装行为。

3. 运行全部测试
- 执行：`python -m pytest -q`（工作目录：仓库根）。
- 若 pytest 不存在或报错：立即停止并报告，不安装。

4. 运行内置脚本测试
- 执行：
  - `python test_basic.py`
  - `python -m tests.test_neural.py`
- 若导入/运行报错：记录并继续到汇总（不安装）。

5. 结果汇总与分析文档
- 内容：
  - 测试统计（passed/failed/xfailed）。
  - 列出失败用例名与错误摘要（真实失败栈信息精简）。
  - 结合失败用例说明当前代码问题点（如“可满足性而非可导性”的局限）。
- 交付：生成一份文本报告（在仓库根目录），命名如 `TEST_RUN_REPORT.md`。
